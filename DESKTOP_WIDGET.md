# Oaki Studio — Desktop Widget

A lightweight Tauri v2 desktop shell that wraps the hosted `/app/widget` route.  
Artists update project progress from a compact, browser-free window on Windows 11.

---

## Architecture

```
Next.js app (hosted)  ←  Tauri WebView2 shell  ←  installed on workstations
      /app/widget                560×720 px
```

The widget is **not** a separate app. It loads the live hosted URL.  
Frontend updates go live immediately — no reinstall required.

---

## One-time setup (developer machine)

### 1 — Install Rust

```powershell
winget install Rustlang.Rustup
```

Restart the terminal, then verify:

```powershell
rustc --version
cargo --version
```

### 2 — Install Visual Studio C++ build tools

Required by Rust on Windows. Download from:  
https://visualstudio.microsoft.com/visual-cpp-build-tools/

Select: **Desktop development with C++**

### 3 — Install Node dependencies

```bash
npm install
```

This adds `@tauri-apps/cli` as a dev dependency.

### 4 — Generate app icons

Place a 1024×1024 PNG of the Oaki logo at `src-tauri/app-icon.png`, then run:

```bash
npm run tauri:icon -- src-tauri/app-icon.png
```

This generates all required icon formats in `src-tauri/icons/`.

## Development

### Step 1 — Start the Next.js app

```bash
npm run dev
```

### Step 2 — Start the Tauri widget (separate terminal)

```bash
npm run tauri:dev
```

The widget window opens at `http://localhost:3000/app/widget`.  
Login, session, and all widget behaviour are identical to the browser.

## Production build

### 1 — Deploy the Next.js app to Vercel

Push the repo to GitHub, then import it in [vercel.com/new](https://vercel.com/new).

Add these environment variables in **Vercel → Settings → Environment Variables**:

| Variable | Where to find it |
|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase → Project Settings → API |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Supabase → Project Settings → API |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase → Project Settings → API (secret) |

After the first deployment, Vercel gives you a URL like `https://oaki-tracking-app.vercel.app`.

### 2 — Set the production URL and build the installer

`OAKI_WIDGET_URL` **must** be set before building — the build will fail with a clear error if it is missing.  
Use the Vercel URL from the step above.

**PowerShell (Windows):**
```powershell
$env:OAKI_WIDGET_URL = "https://oaki-tracking-app.vercel.app/app/widget"
npm run tauri:build
```

**CMD:**
```cmd
set OAKI_WIDGET_URL=https://oaki-tracking-app.vercel.app/app/widget
npm run tauri:build
```

**bash / macOS:**
```bash
OAKI_WIDGET_URL=https://oaki-tracking-app.vercel.app/app/widget npm run tauri:build
```

`tauri:dev` always loads `http://localhost:3000/app/widget` regardless of this variable.

### 2 — Locate the installer

```
src-tauri/target/release/bundle/
  nsis/   →  Oaki-Studio-Widget_0.1.0_x64-setup.exe   ← recommended
  msi/    →  Oaki-Studio-Widget_0.1.0_x64_en-US.msi
```

### 3 — Version the installer

Rename before distributing:

```
Oaki-Studio-Widget-0.1.0.exe
```

Store in the shared folder:

```
Oaki Studio / Internal Tools / Widget Installers /
```

## Workstation installation

1. Copy the installer to the workstation (USB, shared drive, or network share)
2. Double-click `Oaki-Studio-Widget-0.1.0.exe`
3. The installer runs **per-user** — no admin rights needed
4. Creates Start Menu shortcut: **Oaki Studio Widget**
5. Optionally creates Desktop shortcut
6. Uninstall via Windows Settings → Apps

---

## WebView2 requirement

Windows 11 includes WebView2 by default.  
If a workstation shows a blank window or error:

1. Download: https://developer.microsoft.com/microsoft-edge/webview2/
2. Install **Microsoft Edge WebView2 Runtime** (Evergreen Bootstrapper)

---

## Session behaviour

- Login persists across restarts — users sign in once per workstation
- Supabase stores the session in WebView2's localStorage
- Closing and reopening the widget does not require re-login

## File structure

```
src-tauri/
  src/
    main.rs           Entry point (release: no console window)
    lib.rs            Window setup, URL configuration
  icons/              Generated by `npm run tauri:icon`
  dist/               Placeholder (app always loads remote URL)
  capabilities/
    default.json      Tauri permission config
  Cargo.toml
  build.rs
  tauri.conf.json     Window, bundle, installer settings
```

## Security

- The widget uses normal Supabase browser auth
- `SUPABASE_SERVICE_ROLE_KEY` is **never** included in the desktop build
- All sensitive logic remains on the server
- CSP is disabled (`"csp": null`) since we trust our own hosted app
